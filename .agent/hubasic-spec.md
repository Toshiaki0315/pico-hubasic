# Specification: Hu-BASIC Compatibility Layer (RP2350)

このファイルは、pico-basic が準拠すべき Hu-BASIC の言語仕様、内部構造、および Raspberry Pi Pico 2 (RP2350) 上での実装方針を定義します。

## 1. プロジェクト概要・基本仕様
- **ターゲットハードウェア**: Raspberry Pi Pico 2 (RP2350) / 2.8インチ SPI LCD (Waveshare)
- **基本互換性**: SHARP X1 turbo用 Hu-BASIC (CZ-8FB02) のコマンド体系をメインターゲットとする。
- **基本方針**: Hu-BASICの構文をRP2350上で直接実行し、グラフィック・サウンド・ファイルI/OをRP2350のハードウェア資源にマッピングする。
- **非推奨・除外機能**: Z80 CPUエミュレーション、漢字処理機能（KMODE等）、スーパーインポーズ機能（TVPW, CRT, CHANNEL, VOL, SCROLL等）は実装から完全に除外する。
- **プログラム行番号**: 1 ～ 65535。プログラムは常に行番号の昇順で保持される。

## 2. 変数とデータ型
変数名は「英字1文字」または「英字+数字の2文字」（例: `A`, `B1`）。配列は `DIM` 命令により定義し、多次元配列をサポートする。
- **整数型 (`%`)**: 16ビット符号付き。
- **単精度実数 (`!`)**: 内部実装はC++の標準的な `float` (IEEE 754) で代用・近似してよい。デフォルト型。
- **倍精度実数 (`#`)**: 内部実装はC++の標準的な `double` (IEEE 754) で代用・近似してよい。
- **文字列型 (`$`)**: 最大255文字。

## 3. メモリ管理と仮想化
Pico 2の潤沢なSRAM (520KB) を活かし、論理空間と物理空間を分離して管理する。

### 3.1 論理メモリマップ (BASICエンジンから見える64KB空間)
- `0x0000 - 0x7FFF`: BASIC テキスト領域 (32KB - 拡張可能)
- `0x8000 - 0x9FFF`: 変数・配列格納領域
- `0xA000 - 0xBFFF`: 文字列バッファ領域
- `0xC000 - 0xDFFF`: フリーエリア（NEWON命令で動的に変動）
- `0xE000 - 0xFFFF`: システムワークエリア・スタック

### 3.2 VRAM仮想化・ハードウェア領域
論理64KB空間とは**別に**、ネイティブヒープ上にグラフィックVRAMを確保する。
- **仮想VRAM**: 合計96KB（48KB×2バンク）を管理。
- SPIディスプレイへの描画は仮想VRAMをバッファとし、PRW命令の優先順位（テキストとグラフィックの重なり順マスク）をシミュレートして転送する。

## 4. 中間言語 (Token) 定義
プログラム保存・実行時のトークン化ルール（※実装の進行に合わせて適宜拡張すること）。
- `0x80`: `LIST`
- `0x81`: `RUN`
- `0x82`: `PRINT` / `?`
- `0x83`: `IF`
- `0x84`: `THEN`
- `0x85`: `FOR`
- `0x86`: `NEXT`
- `0x87`: `GOTO`
- `0x88`: `GOSUB`
- `0x89`: `RETURN`

## 5. エラーコードとメッセージ対応表
標準として以下のエラーコードを実装し、適宜呼び出すこと。
- `2`: Syntax error (文法不正)
- `5`: Illegal function call (引数不正)
- `6`: Overflow (数値オーバーフロー)
- `7`: Out of memory (メモリ不足)
- `8`: Undefined line number (存在しない行番号)
- `9`: Subscript out of range (配列添字の範囲外)
- `10`: Duplicate definition (配列の二重定義)
- `13`: Type mismatch (型不一致)
- `14`: Out of string space (文字列領域不足)
- `24`: Bad file mode (ファイルモード不正)
- `26`: File not found (ファイル不在)
- `34`: Reserved feature (NEWONで削除された命令の実行)

## 6. 主要コマンド・ステートメント一覧
エージェントは以下の引数構成（シグネチャ）に厳密に従って構文解析器を実装すること。

### 6.1 システム・制御・編集
| コマンド | 引数構成 | 挙動の概要 |
| :--- | :--- | :--- |
| **INIT** / **CLEAR** | `[アドレス]` | メモリ上限設定、変数・スタックの初期化 |
| **NEWON** | `n (0-9)` | 指定レベルに応じ、特定コマンド群を削除してフリーエリアを動的に確保 |
| **WIDTH** | `c, l, g, d` | 文字数、行数、解像度、表示モードの設定 |
| **CONSOLE** | `ys, yl, xs, xl` | テキスト表示・スクロール領域の制限設定 |
| **CLS** | `[n (0-4)]` | テキスト/グラフィック画面の消去 |
| **FOR** | `v=s TO e [STEP i]` | 指定回数のループ実行 |
| **IF...THEN** | `条件式 THEN 処理 [ELSE 処理]` | 条件分岐。入れ子（ネスト）構造をサポート |
| **GOTO** | `行番号/ラベル` | 指定先への無条件ジャンプ |
| **GOSUB** | `行番号/ラベル` | サブルーチンの呼び出し |
| **REPEAT** | `処理 UNTIL 条件式` | 条件が満たされるまで繰り返す（後判定ループ） |

### 6.2 拡張ハードウェア・I/O系 (Pico 2特化)
| コマンド | 引数構成 | 挙動の概要 |
| :--- | :--- | :--- |
| **GET** | `変数名` | キー入力を1文字取得（待機しない）。USBホスト経由のHIDキーボード入力を想定 |
| **FILES** | なし | SDカード内のファイル一覧表示 |
| **GPIO** | `PIN, MODE, VALUE` | ハードウェアピンの直接制御 (Pico 2独自仕様) |

### 6.3 グラフィック
グラフィックコマンドは常に `WINDOW` 命令で設定されたカレントな座標系を参照すること。

| コマンド | 引数構成 | 挙動の概要 |
| :--- | :--- | :--- |
| **WINDOW** | `(xs,ys)-(xe,ye), (x1,y1)-(x2,y2)` | 画面座標とユーザー座標（スケーリング）の対応付け |
| **PSET** | `(x, y), c` | 点の描画 |
| **LINE** | `(x1,y1)-(x2,y2), m, c, ls, [B/BF]` | 直線、長方形、塗りつぶし長方形の描画 |
| **CIRCLE** | `(x,y), r, c, f, s, e` | 円、楕円、弧、扇形の描画 |
| **POLY** | `(x,y), r, c, step, s, e` | 正多角形、星形の描画 |
| **PAINT** | `(x,y), c, b` | 指定色またはタイルパターンによる領域塗りつぶし |
| **GET@** | `(x1,y1)-(x2,y2), A, c` | 画面イメージを配列変数に読み込む |
| **PUT@** | `(x1,y1)-(x2,y2), A, mode, c` | 配列内のイメージを画面に描画（論理演算対応） |

### 6.4 サウンド
| コマンド | 引数構成 | 挙動の概要 |
| :--- | :--- | :--- |
| **BEEP** | `[n]` | ブザー音の出力設定 |
| **MUSIC** | `"MML文字列"` | 3重和音のMML演奏 |
| **SOUND** | `reg, data` | Pico 2のPWMを用いたAY-3-8910相当のソフトウェアPSG出力 |

## 7. 開発プロセスとアーキテクチャ方針
コード生成エージェントは、以下のルールを厳守して実装を進めること。

1. **関心の分離**: 言語のコアエンジン（字句解析・構文解析・数式パーサー・評価器）と、ハードウェア依存部分（SPI描画、PWM音声、SDカード、USBキーボード）は、明確にインターフェース (C++仮想関数等) を分けて実装すること。
2. **インクリメンタル開発**: 一度に全機能を実装しないこと。まずはMac/PC上の標準入出力をモックとしたコアエンジン（PRINT, FOR, GOTOのみ等）の構築と単体テストから始める。
3. **継続的リファクタリング**: 機能を追加・拡張するごとに、コードが肥大化・複雑化しないよう定期的にリファクタリングを実施し、モジュール性を維持すること。# Specification: Hu-BASIC Compatibility Layer (RP2350)
