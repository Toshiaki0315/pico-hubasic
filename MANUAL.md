# pico-basic ユーザーマニュアル

pico-basic は、Raspberry Pi Pico 2 (RP2350) 上で動作する、SHARP X1 turbo (CZ-8FB02) 互換のネイティブ BASIC インタプリタです。1980年代のマイコンの操作感そのままに、現代の強力なハードウェア機能（カラー液晶、SDカード、GPIO制御）を扱うことができます。

## 1. 基本的な使い方

pico-basic には、入力したコマンドが即座に実行される「ダイレクトモード」と、行番号を付けてメモリに記憶させる「プログラムモード」の2つの使い方が存在します。

### ダイレクトモード（即時実行）
行番号（数字）を付けずにコマンドを入力し、Enterキーを押すとすぐに実行されます。計算やハードウェアの簡単なテストに便利です。
```basic
PRINT 10 * 20
200
Ready
```

### プログラムモード
行番号（1〜65535）を先頭に付けて入力すると、コマンドは実行されずにメモリに記憶されます。
```basic
10 PRINT "HELLO, PICO!"
20 GOTO 10
RUN
```
* **`LIST`**: 記憶されたプログラムを画面に表示します。
* **`RUN`**: 記憶されたプログラムを行番号の小さい順に実行します。
* **`NEW`** (または **`CLEAR`**): メモリ上のプログラムと変数をすべて消去します。

## 2. 変数とデータ型

変数名は「英字1文字」または「英字1文字＋数字1文字」（例: `A`, `B1`）が使用できます。末尾の記号によって扱うデータの種類が変わります。

| 型 | 記号 | 説明 | 例 |
| :--- | :---: | :--- | :--- |
| **単精度実数** | `!` | デフォルトの型です（記号省略可）。小数を含む数値を扱います。 | `A = 3.14` または `A! = 3.14` |
| **整数** | `%` | 16ビットの符号付き整数（-32768 〜 32767）を扱います。高速な処理が可能です。 | `X% = 256` |
| **倍精度実数** | `#` | より精度の高い小数を扱います。 | `PI# = 3.14159265` |
| **文字列** | `$` | 文字の並び（最大255文字）を扱います。ダブルクォーテーション `"` で囲みます。 | `N$ = "PICO 2"` |

*※配列を使用する場合は、事前に `DIM A(10)` のように宣言が必要です。*

## 3. 制御ステートメント

プログラムの流れを制御するための基本的なコマンドです。

| コマンド構文 | 動作の説明 |
| :--- | :--- |
| **`IF`** 条件 **`THEN`** 処理 **`[ELSE`** 処理**`]`** | 条件を満たした時に `THEN` 以降を実行し、満たさない場合は `ELSE` 以降を実行します。 |
| **`FOR`** 変数=初期値 **`TO`** 終値 **`[STEP`** 増減値**`]`** | 変数を変化させながら、`NEXT` までの処理を繰り返します。 |
| **`NEXT`** **`[`** 変数 **`]`** | `FOR` ループの終端を示します。 |
| **`GOTO`** 行番号 | 指定した行番号へ無条件にジャンプします。 |
| **`GOSUB`** 行番号 | 指定した行番号のサブルーチン（ひとまとめの処理）へジャンプします。 |
| **`RETURN`** | サブルーチンから、`GOSUB` の次の命令へ戻ります。 |
| **`REPEAT`** 処理 **`UNTIL`** 条件 | 条件を満たすまで、`REPEAT` と `UNTIL` の間の処理を繰り返します（後判定ループ）。 |

## 4. 画面・グラフィック制御

pico-basic は、仮想VRAMを通じてSPIディスプレイに高速な描画を行います。

### システム・テキスト制御
* **`CLS [n]`**: 画面を消去します。
* **`CONSOLE ys, yl, xs, xl`**: テキストを表示・スクロールさせる領域を制限します。
* **`WIDTH c, l, g, d`**: 画面の文字数、行数、グラフィック解像度などを設定します。

### 図形描画
グラフィックコマンドは `WINDOW` 命令で設定された座標系に基づきます。
* **`PSET (x, y), c`**: 指定した座標に、カラーコード `c` で点を打ちます。
* **`LINE (x1,y1)-(x2,y2), m, c, ls, [B/BF]`**: 直線、または長方形（B: 枠のみ, BF: 塗りつぶし）を描画します。
* **`CIRCLE (x,y), r, c, f, s, e`**: 円や楕円、扇形を描画します。
* **`POLY (x,y), r, c, step, s, e`**: 多角形や星形を描画します。
* **`PAINT (x,y), c, b`**: 指定した境界色 `b` で囲まれた領域を、色 `c` で塗りつぶします。

### イメージ操作
* **`GET@ (x1,y1)-(x2,y2), A, c`**: 画面の指定した長方形領域を、配列変数 `A` に画像データとして取り込みます。
* **`PUT@ (x1,y1)-(x2,y2), A, mode, c`**: 取り込んだ配列変数 `A` の画像データを画面に描画します。`mode` によって論理演算（XORなど）が可能です。

## 5. サウンド・ハードウェア制御

Raspberry Pi Pico 2の機能を活かした拡張コマンドです。

* **`BEEP [n]`**: スピーカーからブザー音を鳴らします。
* **`MUSIC "MML文字列"`** / **`PLAY "MML文字列"`**: MML（Music Macro Language）で演奏します。`PLAY` は `MUSIC` と同一のエイリアスです。
* **`SOUND reg, data`**: ソフトウェアPSG音源（AY-3-8910相当）のレジスタに直接データを書き込みます。
* **`GPIO PIN, MODE, VALUE`**: Pico 2のハードウェアピン（GPIO）を直接制御します。外部LEDの点灯やセンサーの読み取りに使用します。
* **`GET 変数名`**: キーボードからの入力を待たずに1文字取得します（リアルタイムゲームの操作等に使用）。
* **`FILES`**: SDカードに保存されているファイルの一覧を表示します。

## 6. エラーメッセージ一覧

プログラムの実行中に問題が発生した場合、以下のエラーコードが表示されます。

| コード | メッセージ | 主な原因 |
| :---: | :--- | :--- |
| **2** | `Syntax error` | コマンドの綴り間違いや、文法が間違っています。 |
| **5** | `Illegal function call` | コマンドに渡した数値（引数）が、許容される範囲を超えています。 |
| **6** | `Overflow` | 計算結果が変数の扱える最大値・最小値を超えました。 |
| **7** | `Out of memory` | プログラムや変数が大きすぎ、メモリ（フリーエリア）が不足しています。 |
| **8** | `Undefined line number` | `GOTO` や `GOSUB` で指定した行番号が存在しません。 |
| **9** | `Subscript out of range` | 配列変数の指定が `DIM` で宣言したサイズを超えています。 |
| **10** | `Duplicate definition` | すでに宣言済みの配列変数を、再度 `DIM` で宣言しようとしました。 |
| **13** | `Type mismatch` | 数値を入れる変数に文字列を入れようとしたなど、型が一致していません。 |
| **26** | `File not found` | 読み込もうとしたファイルがSDカード上に見つかりません。 |
| **34** | `Reserved feature` | `NEWON` コマンドで削除（無効化）された命令を実行しようとしました。 |
